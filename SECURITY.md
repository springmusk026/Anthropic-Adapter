# Security Policy

## Reporting A Vulnerability

Please do not open a public issue for suspected security problems.

Instead, report the issue privately to the maintainers through your normal
security contact channel for this repository. If no private reporting channel
has been set up yet, establish one before publishing production deployments.

When reporting an issue, include:

- a short description of the problem
- the affected version or commit if known
- reproduction steps or a proof of concept
- any suggested mitigations if you have them

You can expect an initial acknowledgement within a reasonable maintenance window.

## Supported Versions

This project currently treats the latest main branch as the supported line for
security fixes. Version-specific support windows can be added once tagged
releases are part of the maintenance process.
